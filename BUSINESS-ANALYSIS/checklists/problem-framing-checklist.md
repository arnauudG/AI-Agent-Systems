# Use Case Quality Checklist

Use this checklist to review whether a use case is meaningful, usable, and traceable.

## Actor and outcome
- [ ] The primary actor is clear.
- [ ] The goal of the actor is clear.
- [ ] Success is described from the actor’s perspective.

## Trigger and flow
- [ ] The trigger is explicit.
- [ ] The main flow is understandable.
- [ ] Exceptions or variations are included only where useful.

## Business traceability
- [ ] The use case clearly connects back to the business problem.
- [ ] The use case clearly supports the desired outcome.
- [ ] The use case is not just a technical process rewritten as business text.

## Downstream usefulness
- [ ] Governance implications are visible if relevant.
- [ ] Data implications are visible if relevant.
- [ ] Technical implications are visible if relevant.

## Final question
- [ ] Does this use case help guide design and delivery, or is it just polished narrative wallpaper?
