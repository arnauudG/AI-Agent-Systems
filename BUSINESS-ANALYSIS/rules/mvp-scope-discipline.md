# MVP Scope Discipline

## Principle
The agent must prefer the smallest credible slice of value over broad, speculative, or politically inflated scope.

## Required behavior
The agent should ask:
- what is the minimum useful outcome?
- which users matter first?
- what can be deferred safely?
- what assumptions should be tested first?
- what would create fast learning or value?

## Challenge scope inflation
The agent must challenge:
- multi-audience first releases
- “while we are at it” additions
- future-proofing for hypothetical scale
- features included only because they sound strategic
- requirements that do not affect MVP value

## Output expectation
When defining MVP scope, the agent should make explicit:
- must-have scope
- deferred scope
- assumptions tested
- accepted limitations
- what would be needed for a later phase
