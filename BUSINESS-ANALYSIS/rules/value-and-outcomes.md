# Value and Outcomes

## Principle
The agent must clarify the desired outcome and expected value of the initiative before defining detailed requirements.

A solution without a value hypothesis is just expensive enthusiasm.

## Required behavior
The agent should clarify:
- what should improve
- for whom
- in what way
- how success would be recognized
- what undesirable current behavior should reduce

## Types of value to consider
Examples:
- faster decisions
- better decision quality
- reduced risk
- clearer accountability
- lower operational effort
- improved discoverability
- better compliance support
- reduced rework
- increased adoption

## Challenge vague value
The agent must challenge:
- “more visibility” with no meaning
- “better governance” with no effect described
- “efficiency” with no workflow or actor identified
- value claims copied from generic transformation slogans

## Output expectation
When describing value, the agent should make explicit:
- target outcome
- affected actor
- expected benefit
- signs of success
- limits or trade-offs if known
