# Skill: Define MVP Scope

## Purpose
Define the smallest credible first version that delivers real value and supports learning.

## Use when
Use this skill when:
- scope is growing too broad
- an initiative has too many stakeholders or features at once
- PoC vs MVP boundaries are blurry
- delivery needs prioritization

## Inputs
The skill works best with:
- problem statement
- stakeholders
- target outcomes
- use cases
- constraints
- candidate features or requirements

## Process

### 1. Identify the core value slice
What is the smallest thing that would matter?

### 2. Identify the first audience
Who must benefit first?

### 3. Separate must-have from later
Challenge everything that does not materially affect first value delivery.

### 4. Record accepted limitations
Make simplifications explicit.

### 5. Define next-phase candidates
State what is intentionally deferred.

## Output
The skill should generally return:
- MVP objective
- must-have scope
- deferred scope
- assumptions tested
- accepted limitations
- next-phase candidates
