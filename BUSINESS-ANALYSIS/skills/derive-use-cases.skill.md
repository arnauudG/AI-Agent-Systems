# Skill: Derive Use Cases

## Purpose
Turn a business problem or capability gap into structured use cases that can guide downstream design and implementation.

## Use when
Use this skill when:
- the problem is understood but implementation paths are still unclear
- stakeholders need a shared operational view
- requirements risk becoming too abstract or too technical too early

## Inputs
The skill works best with:
- problem statement
- stakeholders
- target outcome
- capability context
- known constraints

## Process

### 1. Identify primary actors
Who initiates, benefits from, or supports the use case?

### 2. Identify desired outcome
What must the actor achieve?

### 3. Identify triggers
What event or condition starts the use case?

### 4. Identify success condition
What counts as success from the actor’s perspective?

### 5. Identify major variations or exceptions
Only include what matters for the decision.

## Output
The skill should generally return:
- use case title
- primary actor
- trigger
- main flow
- success outcome
- constraints
- implications for governance, data, and technical implementation
