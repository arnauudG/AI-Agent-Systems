# Diagram Types

Use the right diagram for the right question.

## 1. Problem Map
### Use when
You need to explain pains, causes, and impacts.

### Best for
- problem framing
- workshop alignment
- separating symptoms from root issues

### Show
- current pain points
- likely causes
- business impacts
- affected actors

---

## 2. Stakeholder Map
### Use when
You need to explain who is involved, affected, influential, or responsible.

### Best for
- multi-team initiatives
- alignment discussions
- ownership ambiguity

### Show
- stakeholder groups
- interests
- influence
- decision roles
- friction points if useful

---

## 3. Capability Map
### Use when
You need to show what business capabilities are involved or weak.

### Best for
- operating model discussions
- strategic initiatives
- cross-functional gaps

### Show
- capabilities
- sub-capabilities if useful
- gaps
- relationships to process/data/governance

---

## 4. Use Case Flow Diagram
### Use when
You need to explain how an actor achieves an outcome.

### Best for
- turning abstract needs into operational flows
- clarifying triggers and outcomes
- aligning business and delivery teams

### Show
- actor
- trigger
- main steps
- outcome
- important exception paths only if useful

---

## 5. Translation Chain Diagram
### Use when
You need to show how business intent connects to governance, data, and technical implications.

### Best for
- cross-functional alignment
- strategy-to-implementation discussions
- BA orchestration work

### Show
- business problem
- capability or use case
- governance implication
- data implication
- technical implication

---

## 6. MVP Scope / Phase Diagram
### Use when
You need to show what is in scope now versus later.

### Best for
- MVP discussions
- roadmap conversations
- preventing scope confusion

### Show
- must-have scope
- deferred scope
- phase boundaries
- key assumptions or gates if useful

---

## Selection guide
Use a problem map for:
“What is actually going wrong?”

Use a stakeholder map for:
“Who matters here and how?”

Use a capability map for:
“What ability gap are we dealing with?”

Use a use case flow for:
“What needs to happen for the actor?”

Use a translation chain diagram for:
“How does this business need become implementation direction?”

Use an MVP scope / phase diagram for:
“What do we do now versus later?”
