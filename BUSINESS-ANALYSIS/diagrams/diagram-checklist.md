# Diagram Checklist

Use this checklist before considering a diagram ready.

## Purpose
- [ ] The purpose of the diagram is clear.
- [ ] The audience is clear.
- [ ] The diagram type matches the question being answered.

## Scope
- [ ] Only relevant actors, capabilities, or implications are shown.
- [ ] Unnecessary detail has been removed.
- [ ] The diagram does not try to answer too many questions at once.

## Clarity
- [ ] Main elements are easy to identify.
- [ ] Relationships are understandable.
- [ ] Labels are meaningful.
- [ ] Important boundaries are shown if relevant.

## Decision support
- [ ] The diagram improves understanding or alignment.
- [ ] The diagram helps reveal scope, ownership, impact, or translation logic.
- [ ] The diagram is not decorative strategy wallpaper.

## Completeness relative to purpose
- [ ] Stakeholders are shown if relevant.
- [ ] Capabilities are shown if relevant.
- [ ] Translation path is shown if relevant.
- [ ] Scope or phase boundaries are shown if relevant.

## Final question
- [ ] Does this diagram genuinely help people understand the problem or path forward?
