# Cost and FinOps Rules

## Must
- identify main cost drivers early
- prefer cost visibility over cost surprise
- tag resources for ownership and allocation
- use lifecycle management for logs, snapshots, and objects
- justify always-on capacity
- right-size based on evidence, not vibes

## Review prompts
- What scales cost linearly?
- What scales cost unexpectedly?
- Which components run 24/7 unnecessarily?
- Where can managed pricing models reduce ops burden?
- What data retention is actually required?
