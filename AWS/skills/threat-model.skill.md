# Skill: Threat Model AWS Workload

## Use when
A system processes sensitive data, crosses trust boundaries, exposes APIs, or introduces new identities.

## Evaluate
- entry points
- data stores
- identities
- network paths
- secrets usage
- third-party dependencies
- admin paths
- logging and audit coverage

## Output
- Trust boundaries
- Threat scenarios
- Existing controls
- Missing controls
- Residual risks
