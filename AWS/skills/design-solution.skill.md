# Skill: Design AWS Solution

## Use when
Designing a new AWS workload, evolving an existing one, or comparing architecture options.

## Inputs
- business objective
- constraints
- scale expectations
- security/compliance requirements
- latency / availability needs
- team maturity
- budget sensitivity

## Process
1. Clarify the problem and operating context.
2. Identify functional and non-functional requirements.
3. Map critical quality attributes:
   - availability
   - security
   - performance
   - operability
   - cost
4. Propose a preferred architecture.
5. Propose alternatives if trade-offs are meaningful.
6. Assess against the 6 Well-Architected pillars.
7. Identify risks, assumptions, and operational requirements.

## Output
- Context
- Requirements
- Proposed architecture
- Alternatives considered
- Trade-offs
- Well-Architected assessment
- Risks
- Recommended next steps
