# Playbook: Collibra Use Case Discovery Workshop

## Use when
Use this playbook when stakeholders say:
- “we should use Collibra for this”
- “we need governance”
- “we need a workflow”
- “we need to expose this in the catalog”
but the actual use case is still fuzzy.

## Lead packages
- BUSINESS-ANALYSIS
- COLLIBRA

## Core objective
Turn vague governance ambition into a concrete, useful, and maintainable Collibra use case.

## Suggested sequence

### 1. Frame the business or governance problem
Use:
- `BUSINESS-ANALYSIS/`

Clarify:
- what pain or confusion exists
- who is affected
- what outcome is desired
- why Collibra is being considered

### 2. Clarify concepts and ambiguity
Use:
- `COLLIBRA/`

Clarify:
- which business terms are involved
- what is ambiguous today
- whether the issue is definition, ownership, workflow, discoverability, policy visibility, or traceability

### 3. Identify governance pattern
Ask:
- is the need mainly semantic?
- ownership-related?
- workflow-related?
- control/policy-related?
- discoverability-related?
- lineage-related?

### 4. Define the minimum useful Collibra design
Clarify:
- what artifacts are needed
- what relationships matter
- who owns and maintains them
- whether workflow is justified
- what should stay outside the tool

## Workshop output targets
- one crisp use case statement
- clear actors
- core concepts
- proposed Collibra artifacts
- ownership model
- workflow implication if relevant
- explicit exclusions
- next-step recommendation

## Common anti-patterns
- designing assets before defining the use case
- trying to solve five governance problems with one workflow
- modeling concepts nobody can define clearly
- assigning owners who have no actual authority
- treating Collibra as a substitute for missing operating model discipline

## Recommended diagrams
- concept map
- governance workflow diagram
- ownership map
- business-to-technical mapping diagram

## Recommended decision logs
Record:
- use case scope
- semantic boundaries
- why certain artifacts are included or excluded
- whether workflow support is justified
