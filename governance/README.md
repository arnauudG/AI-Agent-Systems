# Governance

Repository-level governance artifacts used to keep this reasoning system maintainable as it evolves.

This folder is not a specialist delivery package like `AWS/` or `DATA/`.
It exists to govern the repository itself.

Use `governance/` for:
- contribution guidance
- repo review criteria
- versioning rules
- changelog discipline
- package evolution ADRs

Typical files:
- `CONTRIBUTING.md` for contribution guardrails
- `review-checklist.md` for reviewing repo changes
- `VERSIONING.md` for version increment rules
- `CHANGELOG.md` for notable repository changes
- `package-evolution-adr-template.md` for structural repo decisions
