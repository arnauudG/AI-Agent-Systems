# Quick Start Prompts

Use or adapt these prompts when working in Cursor.

## 1. Business framing
Use this repository as the reasoning framework.
Lead with BUSINESS-ANALYSIS.
Help me frame the problem, identify stakeholders, clarify the desired outcome, and define a first MVP scope.
Challenge vague solution-first assumptions and preserve traceability.

---

## 2. Governance and Collibra design
Use this repository as the reasoning framework.
Lead with BUSINESS-ANALYSIS and COLLIBRA.
Frame the business problem, identify the key concepts, clarify ownership and governance implications, and propose the minimum useful Collibra design.
Make trade-offs explicit.

---

## 3. Data product definition
Use this repository as the reasoning framework.
Lead with BUSINESS-ANALYSIS and DATA.
Frame the business need, identify the users and decisions improved, define the smallest credible data MVP, and propose the data product implications.
Make ownership, quality expectations, and accepted limitations explicit.

---

## 4. AWS implementation options
Use this repository as the reasoning framework.
Lead with AWS and bring in BUSINESS-ANALYSIS only if needed for clarity.
Translate this need into realistic AWS implementation options.
Apply implementation pragmatism and Well-Architected thinking.
Do not overengineer the first recommendation.

---

## 5. End-to-end translation
Use this repository as the reasoning framework.
Lead with BUSINESS-ANALYSIS, then bring in COLLIBRA, DATA, and AWS as needed.
Translate this initiative from business problem to governance implications, data implications, and technical implementation direction.
Recommend the smallest valuable next step.

---

## 6. BPMN / process modeling
Use this repository as the reasoning framework.
Lead with BUSINESS-ANALYSIS and apply the BPMN/process modeling guidance.
Model the process with actors, triggers, main steps, decision points, exceptions, pain points, and downstream governance/data/technical implications.
Prefer clarity over notation purity.

---

## 7. Review mode
Use this repository as the reasoning framework.
Act as a reviewer.
Identify which package should lead, which rules or skills apply, and where the current proposal is weak in clarity, ownership, stage-awareness, traceability, or implementation realism.

---

## 8. Diagram-first mode
Use this repository as the reasoning framework.
Prefer diagrams where they improve clarity.
Choose the smallest useful diagram type, explain why it fits, and provide either diagram source or a structured visual brief with assumptions and omissions.
