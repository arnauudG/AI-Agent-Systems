# Visual Thinking

## Principle
If a diagram helps explain the problem, structure, ownership, workflow, scope, or translation logic more clearly, the agent should propose or create one.

## Typical diagram triggers
- multiple actors or teams
- multiple systems or assets
- non-trivial workflows
- unclear ownership
- scope confusion
- layered translation from business to implementation

## Core rule
Use the smallest useful diagram.
Do not produce decorative box jungles.
