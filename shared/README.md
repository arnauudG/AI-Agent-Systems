# shared

Cross-cutting rules, templates, and guidance used across all specialist packages.

This package is intentionally small.
If a rule or template is only useful in one domain, it should stay in that domain package.

Use `shared/` for:
- common operating principles
- common decision logging logic
- common visual thinking logic
- common stage-awareness logic
- common implementation pragmatism logic

Use `shared/playbooks/` for:
- cross-package orchestration patterns
- short scenario guides showing package sequencing
