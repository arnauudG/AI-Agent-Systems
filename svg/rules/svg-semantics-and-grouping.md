# SVG Semantics and Grouping Rules

## Goal
Reflect diagram meaning in SVG structure.

## Rules
- Each meaningful conceptual block should map to a group.
- Group labels with the shapes they describe.
- Group connectors separately when they represent relationships.
- Distinguish content groups from decorative groups.
- Use IDs that reflect semantics, not just appearance.

## Good examples
- `aws-account-source-team-a`
- `edge-iam-role`
- `resource-policy-note`
- `cross-account-access-arrow`
- `legend-readonly-vs-write`
