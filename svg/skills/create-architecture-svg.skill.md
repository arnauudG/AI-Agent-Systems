# Create Architecture SVG

## Use when
- Designing cloud architecture diagrams
- Representing trust boundaries, systems, access flows, or responsibilities
- Creating presentation-ready SVG manually

## Steps
1. Identify the main conceptual blocks.
2. Define ownership and trust boundaries.
3. Define the primary reading direction.
4. Create a simple layout skeleton before styling.
5. Add labels and supporting annotations.
6. Add connectors only after layout is stable.
7. Group related elements semantically.
8. Review legibility at slide scale.
9. Simplify any unnecessary visual noise.

## Output expectations
- Clean SVG structure
- Meaningful grouping and IDs
- Consistent spacing and typography
- Diagram remains editable by hand
