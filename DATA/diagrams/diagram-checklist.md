# Diagram Checklist

Use this checklist before considering a diagram ready.

## Purpose
- [ ] The purpose of the diagram is clear.
- [ ] The intended audience is clear.
- [ ] The chosen diagram type fits the question being answered.

## Scope
- [ ] The scope is appropriate.
- [ ] Unnecessary detail has been removed.
- [ ] The diagram does not try to explain too many things at once.

## Clarity
- [ ] Main sources / components / outputs are easy to identify.
- [ ] Important boundaries are shown when relevant.
- [ ] Important flows are visible and directional.
- [ ] Labels are understandable and consistent.

## Decision support
- [ ] The diagram helps explain value, flow, ownership, architecture, or lifecycle.
- [ ] The diagram improves discussion or review quality.
- [ ] The diagram is not just decorative technical wallpaper.

## Completeness relative to purpose
- [ ] Ownership is shown if relevant.
- [ ] Data movement is shown if relevant.
- [ ] Lifecycle stages are shown if relevant.
- [ ] Boundaries are shown if relevant.

## Final question
- [ ] Does this diagram genuinely make the work easier to understand?
