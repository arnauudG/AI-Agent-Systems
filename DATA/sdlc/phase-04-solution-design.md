# Phase 04 — Solution Design

## Objective
Design the smallest credible solution aligned with value, maturity stage, and operational reality.

## Key questions
- What is the smallest useful architecture?
- How does data move from source to consumer?
- Where should transformations happen?
- What quality controls are needed now?
- What is intentionally deferred?
- What are the main trade-offs?

## Agent behavior
The agent should:
- prefer simple designs
- compare at least one alternative when useful
- make ownership, quality, and flow explicit
- use diagrams when they improve clarity
- avoid speculative future-platform complexity

## Outputs
Useful outputs include:
- solution design
- data flow diagram
- ownership model
- trade-off summary
- explicit deferred items
