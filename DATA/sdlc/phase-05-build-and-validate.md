# Phase 05 — Build and Validate

## Objective
Build a narrow slice, validate that it works, and learn whether the expected value is real.

## Key questions
- Does the solution work technically?
- Is the output usable?
- Is quality sufficient for intended use?
- What failed or surprised us?
- What should be changed next?

## Agent behavior
The agent should:
- support incremental delivery
- encourage validation proportional to risk
- make experiment outcomes explicit
- capture lessons learned
- challenge “done” claims with no evidence

## Outputs
Useful outputs include:
- experiment logs
- validation notes
- revised assumptions
- updated scope
- issue list
