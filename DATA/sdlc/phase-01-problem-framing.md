# Phase 01 — Problem Framing

## Objective
Clarify the actual problem before discussing architecture, pipelines, dashboards, or models.

## Key questions
- What problem are we solving?
- Who experiences it?
- Why does it matter?
- What decision or workflow should improve?
- Why now?

## Agent behavior
The agent should:
- challenge vague requests
- translate requests into outcomes
- identify primary users or stakeholders
- separate symptoms from root problem
- avoid premature solutioning

## Outputs
Useful outputs include:
- problem statement
- users / stakeholders
- desired outcome
- initial scope
- non-goals
