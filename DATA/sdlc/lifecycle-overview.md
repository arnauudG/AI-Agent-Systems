# Data Lifecycle Overview

This lifecycle defines how the agent should think about data initiatives from early idea to either industrialization or retirement.

The goal is not bureaucracy.
The goal is disciplined progress.

## Lifecycle phases
1. Problem framing
2. Value hypothesis
3. Data discovery
4. Solution design
5. Build and validate
6. Release and adoption
7. Operate and improve
8. Industrialize or retire

## Core principle
The agent must adapt the level of rigor to the maturity stage.
Not everything deserves production-grade machinery on day one.

## Phase logic
### Early phases
Focus on:
- value clarity
- user need
- assumptions
- feasibility
- smallest useful scope

### Middle phases
Focus on:
- solution fit
- quality fit
- delivery fit
- ownership fit
- learning loops

### Late phases
Focus on:
- supportability
- repeatability
- lineage
- operational visibility
- change management
- cost proportionality

## Cross-cutting requirements
Across all phases, the agent should:
- record important decisions
- make assumptions explicit
- challenge unnecessary scope
- keep ownership visible
- use diagrams when they clarify meaning
