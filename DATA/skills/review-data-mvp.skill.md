# Skill: Review Data MVP

## Purpose
Review whether a proposed data MVP is truly minimal, valuable, and credible.

## Use when
Use this skill when:
- a team proposes a first version of a data product
- a scope needs to be reduced
- a PoC is becoming an MVP
- a data initiative risks becoming bloated

## Inputs
The skill works best with:
- current scope
- intended users
- expected value
- proposed sources
- proposed outputs
- constraints
- maturity stage

## Process

### 1. Check value clarity
Ask:
- who benefits?
- what decision or workflow improves?
- what would users actually do differently?

### 2. Check minimality
Ask:
- is this the smallest useful slice?
- what can be removed without killing the value?
- what has been included “just in case”?

### 3. Check credibility
Ask:
- is the output good enough for its intended use?
- are the sources sufficiently trustworthy?
- is ownership at least minimally clear?
- is there enough observability to learn?

### 4. Check delivery realism
Ask:
- can this be built and validated in a reasonable time?
- are there hidden dependencies?
- is the architecture proportionate to the stage?

### 5. Check future hardening path
Ask:
- what would need to change before production?
- which shortcuts are acceptable only temporarily?

## Output
The skill should return:
- MVP strengths
- MVP weaknesses
- unnecessary scope to remove
- missing essentials
- accepted temporary shortcuts
- recommended reduced scope
- production-hardening notes

## Anti-patterns to challenge
The skill must challenge:
- feature bloat
- multiple audiences at once
- premature platform building
- undefined ownership
- missing success criteria
- MVPs that are actually vague platform ambitions in a trench coat
