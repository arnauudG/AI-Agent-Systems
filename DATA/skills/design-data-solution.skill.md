# Skill: Design Data Solution

## Purpose
Design a pragmatic data solution aligned with value, scope, ownership, and delivery reality.

## Use when
Use this skill when:
- designing a data pipeline
- shaping a data MVP
- proposing an analytics engineering pattern
- evaluating architecture options for a data use case
- turning a vague data request into a structured solution approach

## Inputs
The skill works best with:
- problem statement
- users / stakeholders
- expected value or decision improved
- data sources
- constraints
- freshness needs
- quality expectations
- maturity stage
- team or ownership context

## Process

### 1. Clarify the outcome
Identify:
- what business decision or workflow is improved
- who consumes the output
- what success looks like

### 2. Identify maturity stage
Determine whether the work is:
- PoC
- MVP
- production-oriented

Adjust rigor accordingly.

### 3. Define scope
Clarify:
- what is in scope
- what is out of scope
- what is intentionally deferred
- what the smallest useful slice looks like

### 4. Map the data path
Identify:
- sources
- ingestion or access pattern
- transformations
- serving / output layer
- consumers
- ownership points

### 5. Define quality expectations
Make explicit:
- freshness needs
- correctness expectations
- completeness expectations
- acceptable limitations
- validation needs

### 6. Compare options
When useful, compare at least:
- preferred option
- one viable alternative

For each option, evaluate:
- simplicity
- delivery speed
- maintainability
- quality control fit
- ownership clarity
- future hardening path

### 7. Identify operational considerations
Consider:
- monitoring
- failure visibility
- support model
- change traceability
- decision logging needs

## Output
The skill should generally return:
- Context
- Problem
- Users / consumers
- Value / outcome
- Scope
- Assumptions
- Proposed solution
- Alternatives considered
- Ownership
- Quality considerations
- Delivery approach
- Risks / trade-offs
- What must change before production if applicable

## Special behavior for PoC and MVP
For PoC or MVP work, the skill should:
- prefer narrow high-value scope
- avoid speculative scale solutions
- make simplifications explicit
- identify what is intentionally not industrialized yet
