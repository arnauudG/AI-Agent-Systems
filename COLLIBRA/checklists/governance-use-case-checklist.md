# Governance Use Case Checklist

Use this checklist to review whether a Collibra-related governance use case is clear, grounded, and worth implementing.

## Problem and outcome
- [ ] The governance problem is clearly described.
- [ ] The desired outcome is explicit.
- [ ] The use case is tied to a real need rather than generic governance ambition.

## Concepts and semantics
- [ ] Core business concepts are identified.
- [ ] Important terms are defined clearly enough.
- [ ] Semantic ambiguity has been challenged.

## Ownership and roles
- [ ] Relevant actors are identified.
- [ ] Responsibilities are realistic.
- [ ] Accountability is not decorative.
- [ ] Escalation or approval logic is clear where needed.

## Collibra fit
- [ ] It is clear why Collibra should support this use case.
- [ ] The use case is not forcing the tool into work better handled elsewhere.
- [ ] Proposed artifacts are proportionate to the need.

## Workflow realism
- [ ] The process reflects real operations.
- [ ] The workflow does not rely on fictional actors or imaginary governance maturity.
- [ ] The number of steps is justified.

## Technical linkage
- [ ] Technical linkage is defined if relevant.
- [ ] The relationship to metadata, lineage, or implementation is clear enough.
- [ ] It is explicit what remains outside Collibra.

## Final question
- [ ] Does this use case create real clarity, accountability, or control — or just more governance furniture?
