# Asset Model Review Checklist

Use this checklist to review whether a Collibra asset model is coherent, useful, and maintainable.

## Purpose and scope
- [ ] The model supports a real use case.
- [ ] It is clear what the model is meant to represent.
- [ ] Scope is narrow enough to stay understandable.

## Concepts and semantics
- [ ] Asset types map to real business or technical concepts.
- [ ] Important terms are defined clearly.
- [ ] Similar concepts are not being merged carelessly.
- [ ] Semantic precision is proportionate to the use case.

## Structure and relationships
- [ ] Asset types are distinct and justified.
- [ ] Relationships express meaningful business or technical logic.
- [ ] Relationship direction and meaning are clear.
- [ ] The model avoids ornamental complexity.

## Ownership and lifecycle
- [ ] Ownership expectations are explicit.
- [ ] Required stewardship effort is realistic.
- [ ] The model can be updated as the domain changes.
- [ ] Change or approval points are clear where needed.

## Collibra fit
- [ ] The model belongs in Collibra rather than another tool or document.
- [ ] The model does not try to store everything.
- [ ] The implementation burden is justified by expected value.

## Usability
- [ ] Users can understand how to navigate the model.
- [ ] The model supports discovery, action, or accountability.
- [ ] Stewards can maintain it without excessive friction.

## Final question
- [ ] Does this model improve clarity and governance outcomes, or is it mainly a more elaborate diagram of confusion?
