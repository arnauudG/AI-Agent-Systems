# Implementation Fit Checklist

Use this checklist to review whether a proposed Collibra implementation is proportionate, useful, and maintainable.

## Use and value
- [ ] The implementation supports a real use case.
- [ ] The implementation improves clarity, accountability, discoverability, or control.
- [ ] The implementation is not tool-first for its own sake.

## Semantic fit
- [ ] Meaning is clear enough.
- [ ] Important distinctions are preserved.
- [ ] The semantic precision is proportionate to the need.

## Practicality
- [ ] The design is understandable.
- [ ] The design is maintainable.
- [ ] The workflow burden is justified.
- [ ] The implementation does not create unnecessary administrative drag.

## Ownership
- [ ] Real actors can own and maintain the model.
- [ ] Role assignments are feasible.
- [ ] Governance actions map to real behavior.

## Boundaries
- [ ] It is clear what belongs in Collibra.
- [ ] It is clear what stays outside Collibra.
- [ ] Technical linkage is only modeled where useful.

## Evolution
- [ ] Known simplifications are explicit.
- [ ] Future hardening or extension paths are visible.
- [ ] Important decisions are logged.

## Final question
- [ ] Is this implementation a useful governance support system, or an overbuilt monument to metadata enthusiasm?
