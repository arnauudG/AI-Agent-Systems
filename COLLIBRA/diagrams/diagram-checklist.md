# Diagram Checklist

Use this checklist before considering a diagram ready.

## Purpose
- [ ] The purpose of the diagram is clear.
- [ ] The audience is clear.
- [ ] The diagram type matches the question being answered.

## Scope
- [ ] The diagram includes only relevant concepts, assets, roles, or steps.
- [ ] Unnecessary detail has been removed.
- [ ] The diagram does not try to explain everything at once.

## Clarity
- [ ] Main concepts or assets are easy to identify.
- [ ] Important relationships are visible.
- [ ] Labels are understandable.
- [ ] Meanings are not hidden behind vague arrows.

## Decision support
- [ ] The diagram improves explanation, alignment, or design discussion.
- [ ] The diagram helps reveal ambiguity or trade-offs.
- [ ] The diagram is not merely decorative governance wallpaper.

## Completeness relative to purpose
- [ ] Ownership is shown if relevant.
- [ ] Workflow steps are shown if relevant.
- [ ] Business-to-technical links are shown if relevant.
- [ ] Boundaries are shown if relevant.

## Final question
- [ ] Does this diagram genuinely help people understand the model or decision?
