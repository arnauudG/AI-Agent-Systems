# Diagram Types

Use the right diagram for the right question.

## 1. Concept Map
### Use when
You need to explain meanings and distinctions between concepts.

### Best for
- glossary discussions
- semantic ambiguity
- ontology / taxonomy clarification

### Show
- concepts
- key distinctions
- related concepts
- meaningful links

---

## 2. Asset Model Diagram
### Use when
You need to explain how Collibra artifacts are structured.

### Best for
- asset type design
- relationship review
- model simplification discussions

### Show
- asset types
- relationships
- meaningful directions
- supporting classifications if useful

---

## 3. Ownership Map
### Use when
You need to clarify accountability and role structure.

### Best for
- stewardship models
- ownership discussions
- governance operating model alignment

### Show
- roles
- responsibilities
- accountability boundaries
- escalation paths if useful

---

## 4. Workflow Diagram
### Use when
You need to explain a governance process over time.

### Best for
- request / approval processes
- stewardship workflows
- issue management flow
- operating model realism checks

### Show
- actors
- triggers
- steps
- decisions
- outcomes

---

## 5. Business-to-Technical Mapping Diagram
### Use when
You need to show how business concepts connect to governance artifacts and technical assets.

### Best for
- translation discussions
- implementation alignment
- stakeholder alignment across business and technical teams

### Show
- business concepts
- governance artifacts
- technical assets or metadata
- meaningful links between layers

---

## Selection guide
Use a concept map for:
“What does this mean?”

Use an asset model diagram for:
“How is this represented in Collibra?”

Use an ownership map for:
“Who is responsible for what?”

Use a workflow diagram for:
“How does this governance process operate?”

Use a business-to-technical mapping diagram for:
“How does business meaning connect to implementation?”
