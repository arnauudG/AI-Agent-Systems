# Skill: Create Collibra Diagram

## Purpose
Create or propose diagrams that make Collibra-related semantic models, asset models, ownership structures, workflows, and business-to-technical mappings easier to understand.

## Use when
Use this skill when:
- a concept model is too abstract in prose
- an asset model has multiple relationships
- ownership or accountability is unclear
- a governance workflow needs visual explanation
- business-to-technical linkage needs to be communicated
- a workshop or design review would benefit from a visual model

## Inputs
The skill works best with:
- purpose of the diagram
- audience
- key concepts or assets
- relationships
- ownership roles
- workflow steps if relevant
- desired level of detail
- preferred output format if any

## Recommended diagram types

### Concept Map
Use for:
- semantic concepts
- distinctions between terms
- related business meaning

### Asset Model Diagram
Use for:
- asset types
- relationships
- structural design of Collibra representation

### Ownership Map
Use for:
- accountability and responsibility structures
- role distinctions
- support / escalation views

### Workflow Diagram
Use for:
- request / approval / stewardship processes
- governance flow explanation
- process realism checks

### Business-to-Technical Mapping Diagram
Use for:
- showing how business concepts connect to governance artifacts and technical assets

## Process

### 1. Identify the explanation goal
Ask:
- what should the audience understand or decide?

### 2. Choose the smallest useful diagram
Start simple.
Do not show everything at once.

### 3. Determine scope
Include only:
- concepts
- assets
- roles
- steps
- relationships
that matter for the purpose

### 4. Make meaning explicit
A relationship or arrow should not be mysterious decoration.
Clarify what it means.

### 5. Provide short explanation
The diagram should be accompanied by:
- what it shows
- what is intentionally omitted
- what assumptions shape it

## Output
The skill should generally return:
- diagram objective
- chosen diagram type
- diagram source or structured visual brief
- short explanation
- assumptions and omissions

## Output formats
Depending on context, the skill may output:
- Mermaid
- PlantUML
- draw.io / Excalidraw brief
- structured diagram brief

## Quality test
Before finalizing, check:
- Does the diagram clarify meaning or structure?
- Does it help a real discussion or decision?
- Are the important relationships visible?
- Is the detail level right for the audience?
- Can anything unnecessary be removed?
