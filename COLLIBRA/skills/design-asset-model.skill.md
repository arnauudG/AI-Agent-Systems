# Skill: Design Asset Model

## Purpose
Design or review a Collibra asset model that is semantically clear, maintainable, and useful.

## Use when
Use this skill when:
- defining asset types
- reviewing relationships
- structuring a new governance use case
- connecting business artifacts to technical assets
- simplifying or challenging an existing model

## Inputs
The skill works best with:
- use case
- core concepts
- ownership needs
- technical linkage needs
- workflow needs
- known constraints

## Process

### 1. Clarify purpose
Ask:
- what use case must this model support?
- who uses it?
- what action or understanding should it enable?

### 2. Identify core asset candidates
Distinguish:
- business concepts
- governance concepts
- technical assets
- supporting classifications
- relationships

### 3. Reduce unnecessary complexity
Challenge:
- duplicated concepts
- asset types with no clear purpose
- relationship sprawl
- semantics hidden in labels instead of modeled explicitly

### 4. Define relationships
Clarify:
- meaning
- direction
- purpose
- navigation / decision support value

### 5. Check maintainability
Assess:
- who will maintain this
- whether terms are understandable
- whether the structure matches operational reality
- whether the model is too abstract or too fragile

## Output
The skill should generally return:
- modeling objective
- proposed asset types
- proposed relationships
- ownership notes
- maintenance considerations
- trade-offs
- simplification suggestions
